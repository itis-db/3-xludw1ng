# k3-2026
Контрольная работа 3

Задача: Разработка модели данных до 3НФ для OLTP и реализация полнотекстового поиска

1. Необходимо разработать базу данных для своей предметной области

Требования к функциональности:
1. Поддержка полнотекстового поиска по двум текстовым текстовым атрибутам.
2. Поиск должен учитывать морфологию русского языка (использовать словарь russian).
3. Необходимо обеспечить быстрый поиск по частичному совпадению (например, по началу слова) с помощью триграмм.
4. Поиск должен возвращать релевантные результаты, отсортированные по релевантности.

Требования к схемам:
- Каждая таблица должна иметь первичный ключ.
- Все неключевые атрибуты должны зависеть только от первичного ключа (2НФ) и не иметь транзитивных зависимостей (3НФ).
- Добавьте индексы для внешних ключей и полей, которые используются в условиях WHERE, JOIN, ORDER BY.

2. Наполнение таблицы тестовыми данными

Данные должны быть реалистичными, чтобы тестировать поисковые запросы.


3. Реализация поиска по триграммам (pg_trgm)
- Включите расширение pg_trgm.
- Создайте GIN-индекс на основе триграмм (для поддержки поиска по частичному совпадению).
- Напишите запрос для полнотекстового поиска с использованием индекса.

4. Демонстрация
- Продемонстрируйте БД.
- Напишите несколько примеров поисковых запросов с выводом EXPLAIN ANALYZE.
- Подготовьте в https://explain.tensor.ru/plan/ анализ плана запроса.
